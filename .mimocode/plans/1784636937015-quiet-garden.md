# GoNow Android Full Refactor Plan

## Current State

**Git**: Branch `codex/feat/android-full-profile-auth`, 14 commits ahead of remote. 5 untracked files, 5 unstaged modifications, 14 staged files.

**Android Architecture** (47 Kotlin files):
- `core/` - SettingsPrefs, weather, location
- `data/` - AuthRepository, ActivityRepository, SocialRepository, NotificationRepository, TokenStore, DeviceIdentity
- `models/` - Single 791-line `ApiModels.kt` with ALL models
- `network/` - ApiClient, ApiService
- `ui/auth/` - Auth screens
- `ui/main/` - All feature screens mixed together (20+ files)
- `ui/theme/` - Theme, Color, Components, Type

**iOS Architecture** (reference):
- `Core/` - Authentication, DesignSystem, Device, Localization, Location, Media, Networking, Notifications, Profile, Storage, Weather
- `Features/` - Activity, Authentication, Chat, Main/Views, Map, Notifications, Profile/Views, Social, Tasks/Views
- `Models/` - Separate models directory

## Key Problems to Fix

1. **ApiModels.kt (791 lines)** - All models in one file, should be split by domain
2. **ProfileStatus has Compose dependency** (`androidx.compose.ui.graphics.Color`) in a domain model
3. **No feature-based directory structure** - All UI mixed in `ui/main/`
4. **AuthViewModel (282 lines)** handles auth + profile + photos (too many concerns)
5. **MapViewport, MapContentState, ActivityDraft, WizardStep** defined in ViewModel files instead of models
6. **Missing iOS features to port**: DesignSystem, Localization support, proper feature separation
7. **No proper error handling pattern** - try/catch scattered everywhere with `catch (_: Exception)`

## Refactor Plan

### Step 1: Split ApiModels.kt into domain files

Create separate model files under `models/`:
- `models/AuthModels.kt` - Auth requests/responses, TokenSet, AuthData, AuthPhase, ApiError
- `models/UserModels.kt` - User, ProfileStatus, ProfilePhoto, ProfilePhotos, UpdateProfileRequest
- `models/ActivityModels.kt` - GoNowActivity, MapActivityResponse, CreateActivityRequest, all Activity enums
- `models/SocialModels.kt` - SocialUser, Conversation, ChatMessage, MeetingInvitation
- `models/NotificationModels.kt` - GoNowNotification, NotificationFeed, NotificationPreferences
- `models/MapModels.kt` - MapCoordinate, MapBounds, MapFilterState, MapViewport, MapContentState
- `models/ApiModels.kt` - Keep only ApiEnvelope, ApiErrorEnvelope, ApiErrorBody, DevicePayload

**Remove Compose dependency from ProfileStatus**: Move `color` property to a UI extension in `ui/theme/` or use a `@Composable` wrapper.
