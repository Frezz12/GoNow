# 002. Access и refresh tokens

## Контекст

Мобильному клиенту нужна сохраняемая сессия при коротком сроке действия прав доступа.

## Решение

Access token — короткоживущий JWT HS256. Refresh token — криптографически случайная непрозрачная строка, привязанная к устройству.

## Причины

JWT удобен для защищённых запросов; серверная refresh-сессия позволяет отозвать отдельное устройство.

## Последствия

Секреты JWT задаются только окружением. `email_verified` пока не блокирует MVP-вход: доставка почты появится отдельным этапом.

## Будущие изменения

Возможны асимметричная подпись JWT и обязательная верификация email для чувствительных действий.
