# 003. Rotation refresh token

## Контекст

Долгоживущий токен нельзя безопасно хранить и переиспользовать бесконечно.

## Решение

В PostgreSQL хранится только SHA-256-хеш токена с серверным секретом. При refresh прежняя запись отзывается и в той же транзакции создаётся новая.

## Причины

Утечка базы не раскрывает токены, а повторное использование старого токена обнаруживается как отозванная сессия.

## Последствия

Клиент обязан атомарно заменить сохранённые токены и сериализовать refresh-запросы.

## Будущие изменения

Можно добавить семейства токенов и глобальный отзыв при признаке компрометации.
