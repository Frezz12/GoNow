# GoNow map

Технический контракт карты активностей. Клиенты используют MapLibre Native для отрисовки, публичные стили и векторные тайлы OpenFreeMap, географические данные OpenStreetMap и GoNow API для активностей.

## Компоненты

| Компонент | Назначение |
| --- | --- |
| MapLibre Native iOS `6.26.0` | Нативная отрисовка векторной карты и интерактивных слоёв |
| OpenFreeMap | Публичный стиль Liberty и векторные тайлы, получаемые через GoNow backend |
| OpenStreetMap | Географические данные карты |
| GoNow backend | Создание активностей и выборка по видимой области |

Ключ, аккаунт и платёжная карта для текущего публичного OpenFreeMap endpoint не нужны. Клиент явно загружает `${API_BASE_URL}/map/style` без локального HTTP-кэша, проверяет структуру MapLibre Style JSON v8 и только затем создаёт карту. Каждый новый документ стиля получает уникальный параметр запроса, поэтому запуск карты всегда виден в backend-логах и не может незаметно использовать устаревший style response. Backend заранее проверяет стиль вместе с TileJSON, встраивает шаблон векторных тайлов и переписывает URL тайлов, шрифтов и спрайтов на тот же GoNow API. Поэтому DNS/VPN устройства не должен напрямую разрешать `tiles.openfreemap.org`, а частично загруженный Natural Earth без городов и дорог не считается готовой картой.

Если системный DNS backend не разрешает OpenFreeMap из-за VPN, запрос автоматически повторяется через DNS-over-HTTPS Cloudflare. TLS остаётся привязанным к `tiles.openfreemap.org`; отключение проверки сертификата и небезопасные HTTP-адреса не используются.

Если backend не может получить полный OpenFreeMap даже после DoH-повтора, `/map/style` возвращает локально сформированный растровый OpenStreetMap-стиль с улицами, городами и объектами. Карта и метки активностей остаются рабочими. HTTP-загрузка стиля ограничена 20 секундами, а начальный рендер MapLibre после получения документа — 15 секундами. Индикатор исчезает после первого пригодного к использованию кадра. Ошибка отдельного тайла после загрузки стиля не перекрывает уже работающую карту, а полный отказ показывает стеклянное сообщение с кнопкой повторной загрузки.

Карта не переводится обратно в состояние loading при обычном переключении вкладок. Задача тайм-аута запускается для первоначального документа стиля и каждого явного retry; камера и уже загруженные тайлы остаются в MapLibre-кэше.

При первом открытии карты приложение запрашивает разрешение и один раз центрирует камеру на актуальной геопозиции с городским масштабом. После первоначального позиционирования камера не следует за пользователем и не прерывает его жесты. Круглая стеклянная кнопка геолокации сразу центрирует карту на последней известной координате и параллельно уточняет её разовым системным запросом. Запрос ограничен 10 секундами: при проблеме кнопка не остаётся в бесконечной загрузке и доступна для повторной попытки.

Сама кнопка геолокации не обращается к GoNow API: координату безопасно предоставляет системная служба устройства. После готовности карты и каждого завершённого перемещения камеры клиент отправляет видимую область в `GET /api/v1/activities/map`; поэтому в backend-логах после загрузки стиля должен появиться запрос `activities.map`, даже если в области пока нет активностей.

Backend адаптирует Liberty-стиль под карту активностей: использует одно нативное название вместо двухстрочной латинской/локальной подписи, раньше показывает населённые пункты с классификацией `town`/`village`, а также немного раньше проявляет региональные и локальные дороги. Для settlement-слоёв используется OpenMapTiles `rank`, несколько допустимых позиций подписи вокруг точки и уменьшенная collision-зона. Подписи страны и области сохраняют обзорный контекст, но не вытесняют города; область скрывается начиная с zoom `6`. Это особенно важно для России, где в одном тайле одновременно находятся десятки длинных названий регионов и населённых пунктов.

После первого сбоя системного DNS проверенный DNS-over-HTTPS-клиент переиспользуется для следующих тайлов, шрифтов и спрайтов. Backend не ждёт повторного DNS-тайм-аута для каждого нового участка карты, поэтому переход в незакэшированный регион при VPN не должен выглядеть как частичная или волнообразная загрузка.

В production следует учитывать, что публичный OpenFreeMap предоставляется без SLA. Провайдер изолирован в backend-модуле `map_proxy`, поэтому его можно заменить на собственный OpenFreeMap deployment, CDN или совместимое S3-хранилище без обновления мобильных клиентов и без изменения API активностей. Reverse proxy должен передавать `X-Forwarded-Proto` и `X-Forwarded-Host`.

На карте всегда показывается кликабельная атрибуция:

`© OpenStreetMap contributors · OpenFreeMap`

## iOS-архитектура

Модуль `Features/Map` разделён по ответственности:

- `Models` — координаты, viewport, категории, активности, фильтры и состояние;
- `Data` — протокол репозитория, TTL/LRU-кэш и mock только для тестов;
- `Services` — HTTP-адаптер GoNow API и загрузчик/валидатор документа стиля карты;
- `ViewModels` — загрузка, debounce/cancellation, создание и выбор активности;
- `Components` — MapLibre SwiftUI bridge, слои, карточка и фильтры;
- `Views` — карта и существующие weather/profile/search/create overlays.

MapLibre предоставляет UIKit view, поэтому `MapLibreActivityMap` подключает его через `UIViewRepresentable`. Проверенный style JSON передаётся bridge напрямую; сетевой запрос стиля и UI-состояния загрузки остаются под контролем приложения. Активности передаются одним `MLNShapeSource`, а не отдельными SwiftUI view. Источник включает client-side clustering с радиусом `58` до zoom `14`.

Слои:

- `gonow-cluster-halo`, `gonow-clusters`, `gonow-cluster-count` — кластер и количество;
- `gonow-selected-activity-halo`, `gonow-activity-markers` — метки и выбор;
- `gonow-user-location-halo`, `gonow-user-location-point` — текущая геопозиция.

Нажатие на кластер увеличивает карту до expansion zoom. Нажатие на метку открывает карточку. Камера сохраняется локально; запрос видимой области выполняется после остановки карты с debounce `400 ms`. Ответы кэшируются на 60 секунд, максимум восемь областей. Ошибка обновления не удаляет уже показанные метки.

MapLibre Native iOS в этой версии использует плоскую Mercator-карту. Globe projection не является обязательной частью визуального контракта и не имитируется декоративно.

## Создание активности

1. Пользователь нажимает «Создать» на карте.
2. Клиент получает текущую координату через системное разрешение геолокации.
3. Пользователь вводит название и выбирает категорию.
4. Клиент отправляет `POST /api/v1/activities`.
5. Backend сохраняет активность в PostgreSQL и возвращает созданный объект.
6. Клиент сразу добавляет объект в текущий map source, выделяет метку и открывает карточку.
7. Последующая загрузка viewport получает ту же активность через `GET /api/v1/activities/map`.

Без координаты создание недоступно. При запрете геолокации обычный просмотр карты продолжает работать.

## Backend и масштабирование

### Доставка базовой карты

В development текущая схема `клиент → GoNow backend → OpenFreeMap` оправдана: она устраняет зависимость телефона от DNS/VPN-доступности внешнего домена, централизует стиль и позволяет быстро заменить поставщика без выпуска новой версии приложения.

В production не следует передавать большой трафик тайлов через основной API-процесс без CDN. Целевая схема:

1. iOS/Android получают стиль с GoNow API или отдельного map-домена.
2. Стиль указывает тайлы, glyphs и sprites на `maps.gonow...`.
3. Перед map-доменом находится CDN с долгим immutable-кэшем по версии tileset.
4. CDN обращается к отдельному tile proxy/server либо напрямую к собственному PMTiles/MBTiles/S3 origin.
5. Основной backend продолжает обслуживать активности, профили и авторизацию, но не становится узким местом для каждого движения карты.

То есть проксирование через backend сейчас правильно как development/VPN-адаптер. Перед production его нужно вынести за CDN или в отдельный map-сервис, сохранив тот же публичный URL-контракт для клиентов.

Первая версия хранит `latitude` и `longitude` как проверяемые `DOUBLE PRECISION`, поэтому работает с текущим образом `postgres:17-alpine` без расширений. Запрос корректно обрабатывает viewport через antimeridian и ограничен максимум 1 000 объектов.

Это сознательный development-этап. При высокой плотности данных переход выполняется без изменения клиентской модели:

1. PostgreSQL заменяется на PostGIS-образ.
2. Добавляется `geography(Point, 4326)` и GiST index.
3. JSON-выдача остаётся для обычных viewport.
4. Для глобального масштаба добавляются server-side aggregation или MVT.

Изображения не входят в map response; подробности активности и media загружаются отдельными ручками.

## Проверка

```bash
xcodebuild -resolvePackageDependencies \
  -project apps/ios/GoNow/GoNow.xcodeproj \
  -scheme GoNow

xcodebuild test \
  -project apps/ios/GoNow/GoNow.xcodeproj \
  -scheme GoNow \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro'
```

Для ручной проверки: разрешённая/запрещённая геолокация, создание активности, повторная загрузка приложения, светлая/тёмная тема, Reduce Motion, отсутствие сети, быстрый pan карты и раскрытие плотного кластера.
